# 예은교회 암송성구 테스트 (yn-bible-test)

매주 한 절씩 외우는 성구를 재미있고 효과적으로 테스트할 수 있는 웹 애플리케이션입니다.

## 🔗 배포 주소
[https://er9910.github.io/yn-bible-test/](https://er9910.github.io/yn-bible-test/)

## ✨ 주요 기능
1. **랜덤 빈칸 퀴즈**: 성구의 단어들을 랜덤하게 빈칸으로 만들어 암송을 돕습니다.
2. **난이도 조절**:
   - **하 (Easy)**: 빈칸 비율 20%
   - **중 (Normal)**: 빈칸 비율 40%
   - **상 (Hard)**: 빈칸 비율 70%
3. **스마트 채점**: 띄어쓰기, 마침표, 쉼표 등 문장 부호를 무시하고 글자만 맞으면 정답 처리됩니다.
4. **테마 설정**: 라이트 모드, 다크 모드, 시스템 설정 모드를 지원하며 설정이 저장됩니다.
5. **성구 목록**: 지금까지 등록된 모든 성구를 한눈에 확인하고 복습할 수 있습니다.
6. **모바일 최적화**: 스마트폰에서도 쾌적하게 이용할 수 있는 반응형 UI를 제공합니다.

## 🛠 기술 스택
- **Frontend**: HTML5, CSS3 (Vanilla), JavaScript (Vanilla)
- **Data**: JSON (`verses.json`)
- **Deployment**: GitHub Pages

## 📖 운영 규칙 (AI 컨텍스트)
새로운 세션에서 작업을 이어서 할 때 아래 규칙을 반드시 준수하십시오:
1. **주간 기준**: 한 주는 **주일(일요일)**부터 시작하여 토요일에 끝납니다. (`YYYY-MM-DD` 형식의 주일 날짜를 사용)
2. **성경 버전**: 모든 성구는 **개역한글 (KRV)** 버전을 원칙으로 합니다.
3. **데이터 관리**: 새로운 성구는 `verses.json` 배열에 객체 형식으로 추가합니다.
   - 형식: `{"week": "YYYY-MM-DD", "reference": "장 절", "text": "성구 내용"}`
4. **비교 로직**: 사용자가 정답을 입력할 때 공백과 특수문자를 제거한 뒤 비교하는 `normalize` 함수를 유지해야 합니다.

## 📂 파일 구조
- `index.html`: 웹 애플리케이션 메인 로직 및 UI
- `verses.json`: 성구 데이터베이스
- `main.py`: (초기 버전용) CLI 테스트 스크립트
